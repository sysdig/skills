# Onboarding Summary Artifact

After successful verification (Step 8), generate a summary artifact in both
Markdown and HTML formats. This gives the user a tangible deliverable they
can share with their team or archive.

## Markdown Template

Write `onboarding-summary.md` to the working directory using this structure:

```markdown
# Sysdig Onboarding Summary

| Field | Value |
|-------|-------|
| Date | {{DATE}} |
| Provider | {{PROVIDER}} |
| Scope | {{SCOPE}} |
| Account/Project ID | {{CLOUD_ACCOUNT_ID}} |
| Sysdig Region | {{SYSDIG_REGION}} |
| Sysdig Secure URL | {{SYSDIG_SECURE_URL}} |

## Capabilities Enabled

{{CAPABILITIES_LIST — bullet list of enabled capabilities with brief description}}

## Resources Created

{{RESOURCE_COUNT}} resources managed by Terraform ({{STATE_ENTRY_COUNT}} total
entries in state including data sources):

\`\`\`
{{TERRAFORM_STATE_LIST — output of `terraform state list`, filtered to exclude
data.* entries. Use: terraform state list | grep -v '^data\.'}}
\`\`\`

## Sysdig Links

- [Cloud Accounts]({{CLOUD_ACCOUNTS_URL}}) — Check connection status
- [Inventory]({{INVENTORY_URL}}) — Browse cloud asset inventory
{{IF CDR: - [Events]({{EVENTS_URL}}) — View security events (last 24h)}}

## Terraform Configuration

\`\`\`hcl
{{MAIN_TF_CONTENT — the generated main.tf file contents}}
\`\`\`

---
Generated by Sysdig Onboarding Assistant
```

## HTML Generation

After writing the Markdown file, generate `onboarding-summary.html` by
converting the Markdown to a self-contained HTML page. Use a simple
approach with no external dependencies:

1. Read the Markdown content you just wrote (you have it in memory from
   generating it — do NOT re-read the file).
2. Write an HTML file that wraps the content in a styled template.

Use this HTML structure:

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Sysdig Onboarding Summary</title>
  <style>
    body { font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif; max-width: 900px; margin: 2rem auto; padding: 0 1rem; color: #1a1a1a; line-height: 1.6; }
    h1 { color: #0f1419; border-bottom: 2px solid #e1e4e8; padding-bottom: 0.5rem; }
    h2 { color: #24292e; margin-top: 2rem; }
    table { border-collapse: collapse; width: 100%; margin: 1rem 0; }
    th, td { border: 1px solid #d0d7de; padding: 0.5rem 1rem; text-align: left; }
    th { background: #f6f8fa; font-weight: 600; }
    pre { background: #f6f8fa; border: 1px solid #d0d7de; border-radius: 6px; padding: 1rem; overflow-x: auto; font-size: 0.85rem; }
    code { font-family: 'SFMono-Regular', Consolas, monospace; }
    ul { padding-left: 1.5rem; }
    a { color: #0969da; text-decoration: none; }
    a:hover { text-decoration: underline; }
    .footer { margin-top: 3rem; padding-top: 1rem; border-top: 1px solid #e1e4e8; color: #656d76; font-size: 0.85rem; }
  </style>
</head>
<body>
  {{HTML_CONTENT — convert Markdown sections to HTML tags:
    - # → <h1>, ## → <h2>
    - table → <table> with <th>/<td>
    - bullet lists → <ul><li>
    - fenced code blocks → <pre><code>
    - links → <a href="...">
    - footer → <div class="footer">
  }}
</body>
</html>
```

The HTML conversion does NOT need to be a generic Markdown parser. Since you
are generating both the Markdown and the HTML, produce the HTML directly from
the same data you used for the Markdown. Write both files in sequence.

## Rules

- Write both files to the working directory (same directory as the Terraform files).
- The Terraform configuration in the artifact is the **generated** `.tf` file
  content — you already have it in memory from Step 7. Do NOT re-read `.tf`
  files from disk.
- Resource list comes from `terraform state list` output captured during
  Step 8 verification.
- Do NOT include any secrets, tokens, or sensitive values in the artifact.
- If file writes are denied by the user, present the Markdown content in a
  code block.
